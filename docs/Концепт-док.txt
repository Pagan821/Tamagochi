Концепт-документ: "Тамагочи - Поросенок Визенау"  	Антонов Арсений

1. Обзор проекта
"Тамагочи - Поросенок Визенау" - это виртуальный питомец, реализованный как desktop-приложение на Windows Forms. Игроку предстоит заботиться о поросенке породы Визенау, удовлетворяя его ежедневные потребности и следя за его здоровьем.
__________________________________________________________________________________________________________________________________________________________________________________________________________________
2. Основная концепция
2.1 Игровая механика
Виртуальный питомец: Поросенок Визенау с уникальными характеристиками

Система потребностей: 5 основных параметров для отслеживания

Ежедневные задачи: Случайно генерируемые потребности на каждый день

Временная система: Игровые дни, разделенные на тики таймера

2.2 Основные характеристики питомца

private int hunger = 50;        // Голод (0-100%)
private int happiness = 50;     // Счастье (0-100%)
private int cleanliness = 50;   // Чистота (0-100%)
private int energy = 50;        // Энергия (0-100%)
private int health = 100;       // Здоровье (0-100%)
private bool isSick = false;    // Состояние болезни
private int sickDays = 0;       // Дни болезни

3. Система ежедневных потребностей
3.1 Динамические потребности
Каждый день поросенок имеет случайные потребности:

Питание: 2-3 раза в день

Сон: 1-2 раза в день

Игры: 1-2 раза в день

Купание: 1 раз в 2 дня

3.2 Визуализация потребностей
Игрок видит текущий статус потребностей через интерфейс:

День 1: Нужно: еда (0/3), сон (0/2), игра (0/2), купание (0/1)
__________________________________________________________________________________________________________________________________________________________________________________________________________________
4. Игровые действия
4.1 Основные взаимодействия
Кормление (+30% голод, -10% чистота)

Купание (+40% чистота, -5% счастье)

Игры (+25% счастье, -20% энергия, -10% голод)

Сон (+40% энергия, -15% голод, +5% здоровье)

4.2 Особые действия
Лечение: Требуется при болезни, имеет 70% шанс успеха

Управление скоростью: 3 режима скорости игры

Рестарт: Запуск новой игры после смерти питомца
__________________________________________________________________________________________________________________________________________________________________________________________________________________
5. Система здоровья и болезней
5.1 Механика болезни
Причина: Чистота ≤ 20%

Последствия: Постепенное ухудшение здоровья

Лечение: Специальная кнопка с шансом выздоровления

Риск смерти: После 5 дней болезни возрастает вероятность смерти

5.2 Условия смерти
Поросенок умирает при достижении 0% любого из параметров:

Голод

Счастье

Энергия

Здоровье
__________________________________________________________________________________________________________________________________________________________________________________________________________________
6. Визуальная система
6.1 Состояния поросенка
Приложение отображает 7 различных состояний через изображения:

Обычный - нормальное состояние

Больной - при болезни

Сонный - энергия ≤ 20%

Голодный - голод ≤ 20%

Грязный - чистота ≤ 20%

Грустный - счастье ≤ 20%

Счастливый - все параметры ≥ 80%

6.2 Интерфейс
Progress Bars: Визуализация всех параметров

Цветовая кодировка: Зеленый/оранжевый/красный для статусов

Адаптивный дизайн: Поддержка изменения размеров окна

7. Технические особенности
7.1 Архитектура
Таймерная система: 15 тиков = 1 игровой день

Случайная генерация: Динамические потребности каждый день

Обработка состояний: Автоматическое обновление статусов

7.2 Баланс игры

hunger -= 5;
happiness -= 3; 
cleanliness -= 4;
energy -= 2;

8. Уникальные особенности
8.1 Динамическая сложность
Случайные потребности: Каждый день разные требования

Прогрессирующая болезнь: Увеличивающийся риск с каждым днем

Баланс действий: Каждое действие имеет позитивные и негативные эффекты

8.2 Стратегический элемент
Игрок должен балансировать между:

Немедленными потребностями и долгосрочным здоровьем

Разными типами взаимодействий

Риском и вознаграждением при лечении
